# 起步

快速开始一个项目,在这个例子中你可以快速的搭建一个基于spring-boot-netty-starter (opens new window)的TCP服务器

# Maven配置集成

  • 添加Maven依赖

版本问题

由于发布的时候出现的失误,导致2022.0.0-RELEASE版本不可用,请直接使用≥2022.0.1-RELEASE的版本


    <dependency>
        <groupId>site.peaklee.framework</groupId>
        <artifactId>spring-boot-netty-starter</artifactId>
        <version>${Last-version}</version>
    </dependency>

# 添加主启动器

  • 添加一个SpringBoot的主启动类以作为程序的入口

提示

请注意主启动类是SpringSocketApplication而非SpringBoot的SpringApplication


import org.springframework.boot.autoconfigure.SpringBootApplication;
import site.peakleeframework.core.SpringSocketApplication;

@SpringBootApplication
public class MySocketAppServer {
    public static void main(String[] args) {
        SpringSocketApplication.run(MySocketAppServer.class,args);
    }
}

# 添加一个处理器

  • 添加一个处理器,用来处理连接的业务信息

注意

  • 如果使用继承AdapterSessionInboundTypeHandler<T>或AdapterSessionInboundHandler处理器的方式来处理具体的业务信息,那么就必须要在该类上添加@Handler(1)注解

import site.peakleeframework.annotation.Handler;
import site.peakleeframework.handler.adapter.AdapterSessionInboundTypeHandler;
import site.peakleeframework.session.impl.Session;
import lombok.extern.slf4j.Slf4j;

@Slf4j
@Handler(1)
public class MessageHandler extends AdapterSessionInboundTypeHandler<String> {
    @Override
    public void onMessage(Session session, String msg) {
        log.info("message received:{}",msg);
        //todo Business Method Implementation
    }
    
}
