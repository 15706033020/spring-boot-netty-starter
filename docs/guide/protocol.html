# 协议

协议是一个Socket服务器的传输纽带,也是必不可少的一部分,框架中默认使用的是Netty自带的String类型的编解码器.

# Google Protobuf协议

在框架中如果要使用Google Protobuf的协议,需要对此协议有一定了解,下面是一个在框架中使用Google Protobuf的示例.

Example

  • 创建一个.proto文件

syntax = "proto3";
option optimize_for = SPEED;//加快编译速度
option java_package = "example.peaklee.framework.protoc";//将来生成的文件在那个包下
option java_outer_classname = "MessageProtoc"; //将来生成的外部类名,此名称不可以和下文主体名重复

message MyData{
  //demo 消息id
  int32 id =1;
  //demo 需要发送到的id
  int32 sendId =2;
  //demo 签名
  string sign =3;
  //demo 数据
  bytes data = 4;
}
  • 在Google Protobuf的Github (opens new window)下载一个Protobuf编译器,按照官方教程安装
  • 将安装目录下的bin添加到环境变量,并在MyData.proto文件目录下执行以下命令转为java类
$ protoc --java_out=./ ./MyData.proto
  • 在当前目录下就会生成如下文件夹example.peaklee.framework.protoc(实际上是你指定的包名),在这个文件夹下就会存在一个已经生成好名为MessageProtoc.java的类
  • 将这个类复制到使用的框架任意目录下,并为其添加@Handler(value = 1,type=HandlerType.PROTOC,innerClass=MyData.class)的注解
  • 将配置socket.support-protobuf配置为protobuf类型,至此我们已经将Protobuf协议应用到我们的应用程序中了,是不是很简单呢?

# 自定义解码器(Decoder)

这里自定义解码器时和定义Netty中的编解码器一致,下面是一个简单的使用例子,以帮助你更好的理解编解码器

Example

  • 创建一个字节解码器
import io.netty.buffer.ByteBuf;
import io.netty.channel.ChannelHandlerContext;
import io.netty.handler.codec.ByteToMessageDecoder;
import lombok.extern.slf4j.Slf4j;
import site.peaklee.framework.annotation.Handler;
import site.peaklee.framework.enums.HandlerType;

import java.util.List;

@Slf4j
@Handler(value = 1,type = HandlerType.DECODE)
public class MyByteDecoder extends ByteToMessageDecoder {

    @Override
    protected void decode(ChannelHandlerContext channelHandlerContext, ByteBuf byteBuf, List<Object> list) throws Exception {
        //todo 自定义约定协议,解析字节数组
    }
}
  • 这个MyByteDecoder的类同样继承的是Netty的字节数组转码解码器,不同的是,我们不在需要添加到initChannel中,只需要在类上加入Handler注解即可.

# 自定义编码器(Encoder)

这里自定义编码器时和定义Netty中的编码器一致,下面是一个简单的使用例子,以帮助你更好的理解编解码器

Example

  • 创建一个字节解码器
import io.netty.buffer.ByteBuf;
import io.netty.channel.ChannelHandlerContext;
import io.netty.handler.codec.ByteToMessageDecoder;
import lombok.extern.slf4j.Slf4j;
import site.peaklee.framework.annotation.Handler;
import site.peaklee.framework.enums.HandlerType;

import java.util.List;

@Slf4j
@Handler(value = 1,type = HandlerType.ENCODE)
public class MyByteEecoder extends MessageToByteDecoder<Object> {

    @Override
    protected void encode(ChannelHandlerContext channelHandlerContext, Object o, ByteBuf byteBuf) throws Exception {
        //todo 自定义协议,将类转化为字节数组
    }
}
  • 这个MyByteEecoder的类同样继承的是Netty的字节数组转码编码器,不同的是,我们不在需要添加到initChannel中,只需要在类上加入Handler注解即可.

注意

  • 上面两个例子虽然看似差距不大,但是一定要细心哦,他们的注解所添加的type属性是不一样的.
  • 在使用自定义的编解码器时,请配置socket.support-protobuf属性为:CUSTOM.
