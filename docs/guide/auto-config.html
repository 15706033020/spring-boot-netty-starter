# 配置

# 概述

框架中已经做了大量的自动配置属性,以SpringBoot的配置方式自动配置

  • resources/application.yml
  • resources/application.yaml
  • resources/application.properties

# 通用配置

# socket.application-type

  • 类型: site.peaklee.framework.enums.AppType
  • 可选值: SERVER|CLIENT
  • 默认值: SERVER

    这个属性用于描述你所创建的应用程序是一个服务端还是一个客户端,默认配置为服务端

socket:
  application-type: SERVER

# socket.port

  • 类型: java.lang.Integer
  • 可选值: 0-65535
  • 默认值: 7080

    当你的应用程序是一个服务端的时候,这个属性为启动的服务监听端口,相反应用程序作为一个客户端的时候,这个属性为你需要连接向的服务端端口

    这个属性的默认值是: 7080

socket:
  port: 7080

# socket.enable-log

  • 类型: java.lang.Boolean
  • 可选值: true|false
  • 默认值: false

    这个属性用来控制在Netty的Pipeline中是否要添加日志打印的处理器

    这个属性的默认值是: false

socket:
  enable-log: false

# socket.log-level

  • 类型: io.netty.handler.logging.LogLevel
  • 可选值: TRACE|DEBUG|INFO|WARN|ERROR
  • 默认值: INFO

    这个属性用来控制在Netty的Pipeline中日志打印的级别

    这个属性的默认值是: INFO

socket:
  log-level: INFO

# socket.support-protobuf

  • 类型: site.peaklee.framework.enums.ProtoType
  • 可选值: STRING|PROTOBUF|CUSTOM
  • 默认值: STRING

    这个属性用来控制在框架中使用那种协议进行通信,内部已经实现了String的编解码器,如果选用Protobuf作为通信协议,只需要在Protobuf编译的java类上添加@Handler(value =1,type = HandlerType.PROTOBUF,innerClass=xxx.class)注解即可

    这个属性的默认值是: STRING

socket:
  support-protobuf: STRING

# 服务端配置

# socket.server

  • 类型: site.peaklee.framework.config.ServerConfiguration
  • 可选值: -
  • 默认值: Object

    当socket.application-type属性为SERVER的时候,这个属性将生效并为服务端提供配置.

socket:
  server:

# socket.server.socket-type

  • 类型: site.peaklee.framework.enums.SocketType
  • 可选值: WebSocket|Socket
  • 默认值: Socket

    配置当前的服务器为一个普通的TCP服务器还是一个提供Web协议的WebSocket服务器.

    这个属性的默认值是: Socket

socket:
  server:
    socket-type: Socket

# socket.server.enable-idle

  • 类型: java.lang.Boolean
  • 可选值: true|false
  • 默认值: true

    配置当前服务器是否启用心跳,启用后会自动添加心跳处理器.

    这个属性的默认值是: true

socket:
  server:
    enable-idle: true

# socket.server.idle-time

  • 类型: java.lang.Integer
  • 可选值: -
  • 默认值: 60

    当启用心跳处理器时,该属性生效,并控制读写空闲时长.

    这个属性的默认值是: 60

socket:
  server:
    idle-time: 60

# socket.server.reader-idle-time

  • 类型: java.lang.Integer
  • 可选值: -
  • 默认值: 60

    当启用心跳处理器时,该属性生效,并控制读空闲时长.

    这个属性的默认值是: 60

socket:
  server:
    reader-idle-time: 60

# socket.server.writer-idle-time

  • 类型: java.lang.Integer
  • 可选值: -
  • 默认值: 60

    当启用心跳处理器时,该属性生效,并控制写空闲时长.

    这个属性的默认值是: 60

socket:
  server:
    writer-idle-time: 60

# socket.server.idle-unit

  • 类型: java.util.concurrent.TimeUnit
  • 可选值: NANOSECONDS|MICROSECONDS|MILLISECONDS|SECONDS|MINUTES|HOURS|DAYS
  • 默认值: MINUTES

    当启用心跳处理器时,该属性生效,并控制空闲时长的单位.

    这个属性的默认值是: MINUTES

# socket.server.boss-threads

  • 类型: java.lang.Integer
  • 可选值: -
  • 默认值: 1

    配置服务器Netty的Boss线程组的线程数量.

    这个属性的默认值是: 1

socket:
  server:
    boss-threads: 1

# socket.server.work-threads

  • 类型: java.lang.Integer
  • 可选值: -
  • 默认值: 10

    配置服务器Netty的Work线程组的线程数量.

    这个属性的默认值是: 10

socket:
  server:
    work-threads: 10

# socket.server.socket-backlog

  • 类型: java.lang.Integer
  • 可选值: -
  • 默认值: 128

    同Netty中的SO_BACKLOG属性.

    这个属性的默认值是: 10

SO_BACKLOG

用于指定服务端连接队列长度，当服务器连接处理线程全忙时，已完成三次握手的请求会被临时存放在连接队列中等待被accept，队列满后会拒绝新收到的连接请求

如不设置，Linux下该值默认为128左右，对于连接数不太多的场景，默认值就够了，像常见RPC框架的服务端(如dubbo)就没有设置，RocketMq需要支持更高频的连接请求，所以使用了推荐值1024

socket:
  server:
    socket-backlog: 128

# socket.server.socket-reuseaddr

  • 类型: java.lang.Boolean
  • 可选值: true|false
  • 默认值: false

    同Netty中的SO_REUSEADDR属性.

    这个属性的默认值是: false

SO_REUSEADDR

TCP四次挥手的最后阶段，主动发起关闭的一端会处于TIME_WAIT状态，该状态下的socket所用端口无法被复用(默认时间2MSL=4分钟)

SO_REUSEADDR=true就是通知内核，如果端口忙，但socket状态是TIME_WAIT，可以立即重用端口

socket:
  server:
    socket-reuseaddr: false

# socket.server.socket-keepalive

  • 类型: java.lang.Boolean
  • 可选值: true|false
  • 默认值: false

    同Netty中的SO_KEEPALIVE属性.

    这个属性的默认值是: false

SO_KEEPALIVE

TCP有内置的连接保活机制，保活并不是把挂掉的连接整活，而是及时发现并释放无效连接资源，只留下活跃的连接

由于netty提供的IdleStateHandler可以非常方便、灵活的实现心跳维持和会话管理，所以一般不用TCP自带的KEEP_ALIVE,这里默认值即设置为false

socket:
  server:
    socket-keepalive: false

# socket.server.tcp-no-delay

  • 类型: java.lang.Boolean
  • 可选值: true|false
  • 默认值: true

    同Netty中的TCP_NODELAY属性.

    这个属性的默认值是: true

TCP_NODELAY

TCP/IP协议中针对TCP默认开启了Nagle算法，会对包的发送进行限制，至少满足下面任意一个条件才可以发出

1.写缓冲区的字节数超过指定阈值

2.之前发出的所有包都已收到ack

其好处是减少网络开销和发送接收两端的压力，坏处就是存在发送延时，对于延时敏感型、数据传输量比较小的应用，应该选择关闭Nagle算法，关闭方式为设置TCP_NODELAY=true

socket:
  server:
    tcp-no-delay: true

# socket.server.allow-half-closure

  • 类型: java.lang.Boolean
  • 可选值: true|false
  • 默认值: false

    同Netty中的ALLOW_HALF_CLOSURE属性.

    这个属性的默认值是: false

ALLOW_HALF_CLOSURE

一个连接的远端关闭时本地端是否关闭，默认值为False。值为False时，连接自动关闭

为True时，触发ChannelInboundHandler的userEventTriggered()方法，事件为ChannelInputShutdownEvent

socket:
  server:
    allow-half-closure: false

# socket.server.socket-linger

  • 类型: java.lang.Integer
  • 可选值: -1|0|>0
  • 默认值: -1

    同Netty中的SO_LINGER属性.

    这个属性的默认值是: -1

SO_LINGER

关闭Socket的延迟时间，默认值为-1，表示禁用该功能。

-1表示socket.close()方法立即返回，但OS底层会将发送缓冲区全部发送到对端。

0表示socket.close()方法立即返回，OS放弃发送缓冲区的数据直接向对端发送RST包，对端收到复位错误。

非0整数值表示调用socket.close()方法的线程被阻塞直到延迟时间到或发送缓冲区中的数据发送完毕，若超时，则对端会收到复位错误。

socket:
  server:
    socket-linger: -1

# socket.server.auto-register-session

  • 类型: java.lang.Boolean
  • 可选值: true|false
  • 默认值: true

    是否将连接的客户端信息自动注册到SessionManger管理.

    这个属性的默认值是: true

socket:
  server:
    auto-register-session: true

# WebSocket配置

# socket.server.websocket

  • 类型: site.peaklee.framework.config.WebSocketConfiguration
  • 可选值: -
  • 默认值: Object

    当socket.server.socket-type属性为WebSocket时,该配置将生效.

socket:
  server:
    websocket:

# socket.server.websocket.web-socket-path

  • 类型: java.lang.String
  • 可选值: -
  • 默认值: /

    当socket.server.socket-type属性为WebSocket时,该属性配置连接的后缀路径.

socket:
  server:
    websocket:
      web-socket-path: /

# socket.server.websocket.max-content-length

  • 类型: java.lang.Integer
  • 可选值: -
  • 默认值: 65535

    当socket.server.socket-type属性为WebSocket时,该属性配置连接的最大传输上下文.

socket:
  server:
    websocket:
      max-content-length: 65535

# socket.server.websocket.sub-protocols

  • 类型: java.lang.String
  • 可选值: -
  • 默认值: null

    当socket.server.socket-type属性为WebSocket时,该属性配置WebSocket的子协议类型.

# socket.server.websocket.allow-extensions

  • 类型: java.lang.Boolean
  • 可选值: true|false
  • 默认值: true

    当socket.server.socket-type属性为WebSocket时,该属性配置是否忽略扩展,当启用socket.server.websocket.enable-compress时,必须为true.

socket:
  server:
    websocket:
      allow-extensions: true

# socket.server.websocket.max-frame-size

  • 类型: java.lang.Integer
  • 可选值: -
  • 默认值: 10240

    当socket.server.socket-type属性为WebSocket时,该属性配置WebSocket的最大帧.

socket:
  server:
    websocket:
      max-frame-size: 10240

# socket.server.websocket.enable-compress

  • 类型: java.lang.Boolean
  • 可选值: true|false
  • 默认值: true

    当socket.server.socket-type属性为WebSocket时,该属性配置是否启用压缩.

socket:
  server:
    websocket:
      enable-compress: true

# socket.server.websocket.enable-ssl

  • 类型: java.lang.Boolean
  • 可选值: true|false
  • 默认值: false

    当socket.server.socket-type属性为WebSocket时,该属性配置是否wss的安全连接.

socket:
  server:
    websocket:
      enable-ssl: false

# socket.server.websocket.socket-ssl

  • 类型: site.peaklee.framework.config.SocketSSL
  • 可选值: -
  • 默认值: Object

    当socket.server.websocket.enable-ssl属性为true时,该类配置证书信息.

socket:
  server:
    websocket:
      socket-ssl:

# socket.server.websocket.socket-ssl.cert-path

  • 类型: java.lang.String
  • 可选值: -
  • 默认值: -

    当socket.server.websocket.enable-ssl属性为true时,该属性生效,配置证书的路径地址.

socket:
  server:
    websocket:
      enable-ssl: true
      socket-ssl:
        cert-path: your/cert/path

# socket.server.websocket.socket-ssl.password

  • 类型: java.lang.String
  • 可选值: -
  • 默认值: -

    当socket.server.websocket.enable-ssl属性为true时,该属性生效,配置证书的密码.

socket:
  server:
    websocket:
      enable-ssl: true
      socket-ssl:
        password: your cert password

# socket.server.websocket.socket-ssl.ssl-type

  • 类型: site.peaklee.framework.enums.SSLType
  • 可选值: PKCS12|JKS
  • 默认值: PKCS12

    当socket.server.websocket.enable-ssl属性为true时,该属性生效,配置证书的类型.

socket:
  server:
    websocket:
      enable-ssl: true
      socket-ssl:
        ssl-type: PKCS12

# 客户端配置

# socket.server.client

  • 类型: site.peaklee.framework.config.ClientConfiguration
  • 可选值: -
  • 默认值: Object

    当socket.server.application-type属性为CLIENT时,该类配置生效.

socket:
  client:

# socket.server.client.host

  • 类型: java.lang.String
  • 可选值: DOMAIN|IPV4
  • 默认值: localhost

    当socket.server.application-type属性为CLIENT时,该类配置生效,配置需要连接的服务器地址.

socket:
  client:
    host: localhost

# socket.server.client.thread-count

  • 类型: java.lang.Integer
  • 可选值: -
  • 默认值: 10

    当socket.server.application-type属性为CLIENT时,该类配置生效,配置连接的工作组线程数,一般为机器CPU核心数的2倍.

socket:
  client:
    thread-count: 10

# socket.server.client.time-out

  • 类型: java.lang.Integer
  • 可选值: -
  • 默认值: 15000

    当socket.server.application-type属性为CLIENT时,该类配置生效,配置连接的到服务器的超时时间,单位为毫秒.

socket:
  client:
    time-out: 15000

# socket.server.client.auto-retry

  • 类型: java.lang.Boolean
  • 可选值: true|false
  • 默认值: true

    当socket.server.application-type属性为CLIENT时,该类配置生效,配置断开连接后是否启用自动重新连接到服务器.

socket:
  client:
    auto-retry: true

# socket.server.client.retry-count

  • 类型: java.lang.Boolean
  • 可选值: ≥-1
  • 默认值: -1

    当socket.server.application-type属性为CLIENT时,该类配置生效,配置自动重连的次数,-1表示不限制次数.

socket:
  client:
    retry-count: -1
