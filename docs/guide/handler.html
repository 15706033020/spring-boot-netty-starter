# 处理器

处理器这一概念是出自Netty,在Netty中存在两种处理器,一种是入站处理器,还有一种是出站处理器,这两种处理器在本质上是一样的,编解码器本质上也是一种处理器

为了更加方便的处理业务逻辑,框架中封装了两款入站适配处理器,这两款处理器的唯一区别就是在与消息的泛型转化,其余都是一致的

# 入站处理器

提示建议

  • 由于入站适配处理器实现了AdapterSessionManager接口,所以在继承了这些类之后也会拥有对Session的管理操作

# AdapterSessionInboundHandler

  • 这个类继承自Netty的ChannelInboundHandlerAdapter并实现了AdapterSessionManager,同时提供了一套接口,以供开发者在处理业务逻辑的时候实现

# onRegistered(Session session)

  • 继承AdapterSessionInboundHandler的类实现该接口会在客户端注册但是连接尚未完成的时候触发,传递一个客户端的Session作为参数

Example

import lombok.extern.slf4j.Slf4j;
import site.peaklee.framework.annotation.Handler;
import site.peaklee.framework.handler.adapter.AdapterSessionInboundHandler;
import site.peaklee.framework.session.impl.Session;

@Slf4j
@Handler(1)
public class MessageHandler extends AdapterSessionInboundHandler{
    @Override
    public void onRegistered(Session session) {
        log.info("连接尚未完全建立,注册到的SessionId是:{}",session.getId());
    }
}

# AdapterSessionInboundTypeHandler<T>

该处理器的方法和AdapterSessionInboundHandler是一致的,唯一的不同点在于该类需要一个泛型作为消息处理的类型

Example

import lombok.extern.slf4j.Slf4j;
import site.peaklee.framework.annotation.Handler;
import site.peaklee.framework.handler.adapter.AdapterSessionInboundHandler;
import site.peaklee.framework.session.impl.Session;

@Slf4j
@Handler(1)
public class MessageHandler extends AdapterSessionInboundTypeHandler<String>{
    @Override
    public void onMessage(Session session,String msg) {
        log.info("接收到客户端消息,SessionId是:{} -- 接收到的被泛型转化后的消息是:{}",session.getId(),msg);
    }
}

# 出站处理器

在实际的开发过程中出站处理器我们或许并不常用,使用的最多的也是在Inbound处理器的write()方法,所以在框架中并没有对其做过多的封装和处理,那么在框架中应该如何使用呢?

Example

  • 以下是一个在项目中使用出站处理器的例子

import io.netty.channel.ChannelHandlerContext;
import io.netty.channel.ChannelOutboundHandlerAdapter;
import io.netty.channel.ChannelPromise;
import lombok.extern.slf4j.Slf4j;
import site.peaklee.framework.annotation.Handler;
import site.peaklee.framework.enums.HandlerType;

import java.net.SocketAddress;

@Slf4j
@Handler(value = 1,type = HandlerType.OUTBOUND)
public class OutboundHandler extends ChannelOutboundHandlerAdapter {
    @Override
    public void bind(ChannelHandlerContext ctx, SocketAddress localAddress, ChannelPromise promise) throws Exception {
        super.bind(ctx, localAddress, promise);
    }

    @Override
    public void connect(ChannelHandlerContext ctx, SocketAddress remoteAddress, SocketAddress localAddress, ChannelPromise promise) throws Exception {
        super.connect(ctx, remoteAddress, localAddress, promise);
    }

    @Override
    public void disconnect(ChannelHandlerContext ctx, ChannelPromise promise) throws Exception {
        super.disconnect(ctx, promise);
    }

    @Override
    public void close(ChannelHandlerContext ctx, ChannelPromise promise) throws Exception {
        super.close(ctx, promise);
    }

    @Override
    public void deregister(ChannelHandlerContext ctx, ChannelPromise promise) throws Exception {
        super.deregister(ctx, promise);
    }

    @Override
    public void read(ChannelHandlerContext ctx) throws Exception {
        super.read(ctx);
    }

    @Override
    public void write(ChannelHandlerContext ctx, Object msg, ChannelPromise promise) throws Exception {
        super.write(ctx, msg, promise);
    }

    @Override
    public void flush(ChannelHandlerContext ctx) throws Exception {
        super.flush(ctx);
    }
}

注意

  • 注意Handler注解的type属性参数为: OUTBOUND
