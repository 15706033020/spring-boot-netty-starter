# 注解(Annotation)

  • 随着后端生态的越来越完善,注解开发已经成为不可获取的一部分,注解在开发过程中占据了大部分的过程,它可以帮助我们实现快捷开发,在本框架中也封装了多个注解以供事件处理,程序管理使用

# 处理器

  • 这一类型的注解,主要是对处理器的生命周期进行回调,与Interceptor拦截器和Lambda监听器的生命周期基本一致,但是优先级没有它们高,执行顺序依次是Interceptor->Lambda->Annotation

提示

这一类型的注解必须使用在Spring的Bean中才会生效,且方法必须带有特定参数,否则无法通知.

# 标识

  • 这一类型的注解主要起到标识作用,在程序启动阶段有重要作用

# @EnableWebServer

  • 作用域: 类级别
  • 功能: 这个注解仅在主程序的启动类生效,加入这个注解会在启动Socket服务器的同时启动一个Spring-web的容器
  • 参数: 无

注意

如果使用这个注解,必须添加Spring-boot-web的相关依赖,且依赖版本不得低于框架使用版本

# @HandlerScan

  • 作用域: 类级别
  • 功能: 这个注解仅在主程序的启动类生效,加入这个注解会在启动的时候选择性的进行模块加载,而不会进行全局加载Handler
  • 参数:
参数名称 参数类型 是否必须 参数说明
packages java.lang.String[] true 需要加载的Handler的包路径

# @Handler

  • 作用域: 类级别
  • 功能: 这个注解用来标识一个类是一个处理器,这个类的类型必须是一个ChannelHandler的类型,在这一类类型上加上该注解,将自动注入到Pipeline之中
  • 参数:
参数名称 参数类型 是否必须 参数说明
value java.lang.Integer true 该参数用来确定处理器的优先级,数字越小优先级越高
name java.lang.String false 该参数用来设置Handler的加载名称
type site.peaklee.framework.enums.HandlerType false 该参数用来设置Handler的类型
innerClass java.lang.Class<T> false 该参数只有在type属性为PROTOC的时候有效,用来标识protobuf转化的数据标识内部类

提示

默认情况下被它标识的类不是一个单例,会在每个线程创建一个单独的实例,这些实例互不影响,如果需要创建一个共享的单例以节省开销,你需要添加一个ChannelHandler.Sharable注解,并将这个类加入到Spring的IOC容器中进行管理,或实现SpringInject接口

# 依赖注入

  • 这一类型的注解主要是对Spring容器操作的扩展可以实现在非IOC容器中的注入

# @InjectAttr

  • 作用域: 属性级别
  • 功能: 任何一个处理器,拦截器类型的类内部属性如果被标注这个注解将会自动注入标注属性的值,该值的来源是Spring的配置文件
  • 参数:
参数名称 参数类型 是否必须 参数说明
value java.lang.String true 该参数用来标识配置文件的属性表达式,表达式以${开头,以}结束

表达式

  • 请确保属性类型与配置文件属性类型一致,否则无法注入成功
    //该表达式的意思式注入配置文件中socket.port的值给这个属性,如果没有就给默认值 8080
    @InjectAttr("${socket.port:8080}")
    private Integer port;
    
    //该表达式的意思式注入配置文件中socket.arrays的值给这个数组,如果没有就给默认值 8080,8084,8082,表达式想注入数组类型需以,分割
    @InjectAttr("${socket.arrays:8080,8084,8082}")
    private Integer[] arrays;
    
    //该表达式的意思式注入配置文件中socket.list的值给这个List,如果没有就给默认值 8080,8084,8082,表达式想注入List类型需以,分割
    @InjectAttr("${socket.list:8080,8084,8082}")
    private List<Integer> list;

# @InjectBean

  • 作用域: 属性级别
  • 功能: 任何一个处理器,拦截器类型的类内部属性如果被标注这个注解将会自动注入标注属性的值,该值的来源是Spring容器中的Bean
  • 参数:
参数名称 参数类型 是否必须 参数说明
value java.lang.String false 需要注入的Bean名称
require java.lang.Boolean false 这个Bean是否必须注入

提示

  • 默认情况下Bean是被必须注入的,通常会通过类型去查找,如果设置了Bean的名称,会优先以名称查找,查找不到会抛出错误信息,但是不会终止程序的运行
