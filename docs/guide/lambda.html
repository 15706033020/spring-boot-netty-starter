# Lambda

  • Lambda是基于JAVA8的特性而创建出的一系列能够通过Lambda特性及语法糖来实现对事件的处理机制,主要包含一个方法,两个类

# addHandlerListener

  • 这个方法是所有Lambda函数的入口方法,通过在这个方法上添加基于Lambda的监听函数以实现对于事件的处理,
  • 这个方法的添加在应用程序上下文SocketConfigurableContext对象中
  • 该方法有两个参数,一个是事件类型HandlerEvent,一个是回调参数HandlerCallback.
  • 这个方法的事件与Interceptor无二,只是优先级没有Interceptor高,书写比Interceptor方便,所以当同时存在时,会先执行Interceptor后执行Lambda

# HandlerEvent

这个类型的参数决定了我们添加的事件是属于什么类型的,可选的类型如下

  • Registered 连接尚未成功,与Interceptor中SocketRegistered的时机一致
  • UnRegistered 连接已断开,与Interceptor中SocketUnregistered的时机一致
  • Active 连接成功,与Interceptor中SocketActive的时机一致
  • Inactive 连接离线,与Interceptor中SocketInactive的时机一致
  • BeforeMessage 消息读取之前,与Interceptor中SocketBeforeReadMessage的时机一致
  • Message 读取消息,与Interceptor中SocketMessage的时机一致
  • AfterMessage 消息读取之后,与Interceptor中SocketAfterReadMessage的时机一致
  • Complete 消息读取完成,与Interceptor中SocketCompleteMessage的时机一致
  • Idle 心跳空闲,与Interceptor中SocketIdle的时机一致
  • Error 捕获错误,与Interceptor中SocketExceptionCapture的时机一致

# HandlerCallback

这时一个消息传输参数,它根据我们不同的事件类型来绝地为我们提供什么样的参数

  • session 无论是什么样的事件,类型都会传递该参数
  • msg 仅会在BeforeMessage,Message,AfterMessage这三个事件的时候传递该参数,其余事件均为Null
  • idleState 这个参数仅在Idle事件中传递
  • cause 这个参数仅在 Error 事件中传递

# Example

  • 这个例子将说明他们被如何使用

Example


package com.lee.client.handler;

import lombok.extern.slf4j.Slf4j;
import org.springframework.stereotype.Component;
import site.peaklee.framework.context.SocketConfigurableContext;
import site.peaklee.framework.core.spi.SpringCommandLineRunner;
import site.peaklee.framework.enums.HandlerEvent;
import site.peaklee.framework.session.MessageServerCallback;
import site.peaklee.framework.session.impl.ApplicationSession;

import java.util.Scanner;

/**
 * @author PeakLee
 * @version 2023
 * @serial HandlerListener
 * @since 2023/4/11
 */
@Slf4j
@Component
public class HandlerListener implements SpringCommandLineRunner {
    @Override
    public void run(SocketConfigurableContext context) throws Exception {
        context.addHandlerListener(HandlerEvent.BeforeMessage,cb->{
            log.info("HandlerListener-BeforeMessage:{}",cb.getMsg());
        });
        context.addHandlerListener(HandlerEvent.AfterMessage,cb->{
            log.info("HandlerListener-AfterMessage:{}",cb.getMsg());
        });
        context.addHandlerListener(HandlerEvent.Message,cb->{
            log.info("HandlerListener-Message:{}",cb.getMsg());
        });
        context.addHandlerListener(HandlerEvent.Registered,cb->{
            log.info("HandlerListener-Registered:{}",cb.getSession());
        });
        context.addHandlerListener(HandlerEvent.Active,cb->{
            log.info("HandlerListener-Active:{}",cb.getSession());
        });
        context.addHandlerListener(HandlerEvent.Inactive,cb->{
            log.info("HandlerListener-Inactive:{}",cb.getSession());
        });
        context.addHandlerListener(HandlerEvent.UnRegistered,cb->{
            log.info("HandlerListener-UnRegistered:{}",cb.getSession());
        });
        context.addHandlerListener(HandlerEvent.Complete,cb->{
            log.info("HandlerListener-Complete:{}",cb.getSession());
        });
        context.addHandlerListener(HandlerEvent.Idle,cb->{
            log.info("HandlerListener-Idle:{}",cb.getIdleState());
        });
        context.addHandlerListener(HandlerEvent.Error,cb->{
            log.info("HandlerListener-Error:{}",cb.getCause().getMessage());
        });
    }
}
